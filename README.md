# Brick

**Brick** is an English-like DSL for building browser automations. Write plain, readable code in `.brick` files — Brick compiles them into structured runbook steps that execute in the Intragrid automation engine.

```brick
// main.brick

main(target_url: String, report_name: String) -> Table {
  go to @target_url
  click "#login-btn" as "Login"
  fill "Email" with "user@example.com"
  fill "Password" with @password
  press Enter
  wait 2s

  ai "Wait for the dashboard to fully load" -> @ready
  extract table "All records in the results grid" -> @data

  js {
    return document.title;
  } -> @page_title

  save @data as @report_name
  return @data
}
```

## Installation

### CLI

```bash
cd packages/brick-cli
npm link
```

The `brick` binary is now available globally.

### VS Code / Cursor Extension

```bash
cd packages/brick-vscode
npm run package
code --install-extension brick-lang-0.1.0.vsix
cursor --install-extension brick-lang-0.1.0.vsix
```

Or install directly from the `.vsix` file via **Extensions → Install from VSIX** in the editor.

## CLI Usage

```bash
brick new [name]        # scaffold a new .brick file
brick build <file>      # compile to runbook JSON
brick lint <file>       # validate and report errors
brick print <file>      # pretty-print / round-trip check
```

**Example:**

```bash
brick new scrape-report
brick lint scrape-report.brick
brick build scrape-report.brick   # → scrape-report.json
```

## Language Reference

### Functions

```brick
// Entry point — no inputs, no return type
main() { }

// With typed inputs
main(url: String, rows: Number) { }

// With return type
main(url: String) -> Table { }
```

### Variables

```brick
// Assign a literal value to a variable
"https://example.com" -> @base_url
42 -> @max_rows
["a", "b"] -> @items
{ key: "value" } -> @config

// Shorthand assignment inside a function
@counter = 0
```

### Types

```brick
type ReportRow = {
  name: String
  amount: Number
  status: String
}
```

## Step Reference

| Brick syntax | What it does |
|---|---|
| `go to "url"` | Navigate to a URL |
| `click "#sel" as "Label"` | Click an element |
| `fill "Field" with "value"` | Fill a form field (AI-located) |
| `type "#sel" with "text"` | Type into a CSS selector |
| `select "#sel" to "value"` | Choose a dropdown value |
| `press Enter` | Press a keyboard key |
| `wait 2s` / `wait 500ms` | Pause for a duration |
| `scroll ".el" by 300` | Scroll an element |
| `screenshot -> @snap` | Capture a screenshot |
| `ai "instruction" -> @result` | AI-driven action |
| `extract table "desc" -> @data` | Extract a table from the page |
| `extract from @data "query" -> @val` | Pull a value from a variable |
| `@var = value` | Set a variable |
| `save @data as "name"` | Save a table to the database |
| `js { ... } -> @result` | Run JavaScript in the browser |
| `python { ... } -> @result` | Run Python with Playwright access |
| `load excel @file sheet "Sheet1" -> @data` | Load an Excel file |
| `upload @file to "#input" -> @result` | Upload a file |
| `report title: "Name" { jsx }` | Write a report document |
| `return @value` | Return a value from the function |

## Development

### Build all packages

```bash
npm install          # install and link workspace packages
cd packages/brick-core && npm run build
cd packages/brick-cli && npx tsc
cd packages/brick-vscode && npm run bundle
```

### Rebuild after grammar changes

```bash
# In packages/brick-core
npm run grammar      # regenerate grammar.js from grammar.peggy
npx tsc              # recompile TypeScript
cp src/grammar.js dist/grammar.js
```

### Rebuild the VS Code extension

```bash
# In packages/brick-vscode
npm run bundle       # esbuild + copy grammar.js
npm run package      # produce brick-lang-x.x.x.vsix
```

## Architecture

```
brick/
├── packages/
│   ├── brick-core/          # Parser (Peggy PEG grammar) + compiler + linter
│   │   ├── src/
│   │   │   ├── grammar.peggy   ← grammar definition
│   │   │   ├── grammar.js      ← generated by peggy (do not edit)
│   │   │   ├── ast.ts          ← AST node types
│   │   │   ├── compiler.ts     ← AST → RunbookStep[]
│   │   │   ├── serializer.ts   ← RunbookStep[] → .brick source
│   │   │   ├── linter.ts       ← variable scope + validation rules
│   │   │   └── index.ts        ← public API: parse, compile, lint, buildFromSource
│   │   └── dist/               ← compiled output
│   │
│   ├── brick-cli/           # CLI entry point (commander)
│   │   └── src/index.ts        ← brick build / lint / print / new
│   │
│   └── brick-vscode/        # VS Code / Cursor extension
│       ├── src/extension.ts    ← activation, diagnostics, commands
│       ├── syntaxes/           ← TextMate grammar (syntax highlighting)
│       ├── snippets/           ← 19 built-in snippets
│       ├── esbuild.js          ← bundle script
│       └── out/                ← bundled extension + grammar.js
└── package.json             ← npm workspaces root
```

The grammar lives in `grammar.peggy` (PEG format via [Peggy](https://peggyjs.org/)). Changes to the grammar require running `npm run grammar` in `brick-core` to regenerate `grammar.js`.
